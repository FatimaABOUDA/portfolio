Activité 1

Formulaire d'Inscription avec Traitement PHP

I. Introduction

  • Formulaire d'inscription HTML avec des champs pour les informations personnelles.
  • Traitement côté serveur en PHP pour enregistrer les données dans une base de données.

II. Structure du Code

A. Formulaire HTML
  • Condition: Vérification si le bouton "enregistrer" a été soumis.
  • Actions:
    • Inclusion du fichier bd_connexion.php pour la connexion à la base de données.
    • Préparation d'une requête SQL d'insertion.
    • Liaison des valeurs du formulaire à la requête préparée.
    • Exécution de la requête pour enregistrer les données.
B. Traitement PHP
  • Condition: Si le bouton "enregistrer" est soumis.
  • Actions:
    • Connexion à la base de données à l'aide de la fonction connexionPDO().
    • Récupération des départements depuis la base de données pour le menu déroulant.
    • Affichage du formulaire avec des champs pour la tranche d'âge, les loisirs, et le projet de développement web.

III. Points Importants

A. Sécurité
  • Utilisation de requêtes préparées pour éviter les injections SQL.
B. Base de Données
  • La connexion à la base de données est gérée par la fonction connexionPDO() dans bd_connexion.php.
C. Affichage Conditionnel
  • Le formulaire est affiché ou le traitement est effectué en fonction de l'action de l'utilisateur.
IV. Conclusion
  • Un formulaire complet d'inscription avec une logique de traitement côté serveur pour l'enregistrement des données.

API REST - Gestion des Étudiants

But du Projet

Ce projet consiste en une API REST développée en PHP permettant la gestion des étudiants. Elle offre des fonctionnalités CRUD (Create, Read, Update, Delete) pour interagir avec une base de données d'étudiants.

Endpoints Disponibles

  • POST /etudiant.php : Ajouter un nouvel étudiant.
  • GET /etudiant.php : Récupérer tous les étudiants.
  • GET /etudiant.php?id=X : Récupérer un étudiant spécifique (remplacez X par l'ID de l'étudiant).
  • DELETE /etudiant.php?id=X : Supprimer un étudiant spécifique (remplacez X par l'ID de l'étudiant).
  • PUT /etudiant.php?id=X : Mettre à jour un étudiant spécifique (remplacez X par l'ID de l'étudiant).

Utilisation

Pour utiliser cette API, vous pouvez envoyer des requêtes HTTP à ses différents endpoints en utilisant des outils comme Postman, cURL, ou même depuis votre propre application.

Exemple d'utilisation avec Postman

  1. Ajouter un étudiant : POST /etudiant.php avec les données appropriées dans le corps de la requête.
  2. Récupérer tous les étudiants : GET /etudiant.php.
  3. Récupérer un étudiant spécifique : GET /etudiant.php?id=X.
  4. Supprimer un étudiant : DELETE /etudiant.php?id=X.
  5. Mettre à jour un étudiant : PUT /etudiant.php?id=X avec les nouvelles données dans le corps de la requête.

Remarques

  • Ce projet suppose l'utilisation d'une base de données configurée selon les besoins de l'API.
  • Assurez-vous d'avoir les autorisations nécessaires pour effectuer des opérations CRUD.
  • Les réponses de l'API sont au format JSON.

Projet de Gestion d'Utilisateurs

  • Affiche le tableau de bord de l'utilisateur connecté.
  • Vérifie la connexion via les sessions PHP.
  • Affiche le nom de l'utilisateur et propose un lien de déconnexion.

Inscription

  • Permet aux utilisateurs de s'inscrire.
  • Le formulaire capture le nom d'utilisateur, l'e-mail et le mot de passe.
  • Mot de passe hashé avant le stockage dans la base de données.
  • Message de succès affiché après l'inscription.

Connexion

  • Gère le processus d'authentification des utilisateurs.
  • L'utilisateur entre son nom d'utilisateur et son mot de passe.
  • Le mot de passe est comparé avec le hash stocké.
  • Redirection vers le tableau de bord en cas de succès.

Déconnexion

  • Gère la déconnexion des utilisateurs.
  • La session est détruite, l'utilisateur est redirigé vers la page de connexion.

Remarques Générales

  • Utilisation de fichiers de configuration pour la base de données.
  • Styles CSS inclus via le fichier externe style.css.
  • Utilisation de sessions PHP pour le suivi de l'état de l'utilisateur.
  • Stockage sécurisé des mots de passe avec Bcrypt.

Projet de Gestion d'étudiant

Liste des Étudiants

Une application de gestion d'étudiants utilisant une API REST.

  • Liste des étudiants
  • Ajouter un étudiant
  • Supprimer un étudiant
  • Modifier un étudiant

Objectif

Le projet vise à créer une application de gestion d'étudiants en utilisant une API REST pour effectuer des opérations CRUD (Create, Read, Update, Delete). L'application permet d'interagir avec un serveur distant pour récupérer, ajouter, modifier ou supprimer des informations sur les étudiants.

Structure du Projet

Le projet est divisé en plusieurs pages, chacune offrant une fonctionnalité spécifique :

  • Liste des Étudiants (`read.php`): Affiche une liste de tous les étudiants existants. Permet également de filtrer les étudiants par ID à l'aide d'un formulaire.
  • Ajouter un Étudiant (`create.php`): Permet d'ajouter un nouvel étudiant en saisissant les détails dans un formulaire. Les données sont envoyées au serveur via une requête POST à l'API.
  • Modifier un Étudiant (`update.php`): Affiche la liste des étudiants avec un formulaire de modification associé à chaque étudiant. Lorsque le formulaire est soumis, une requête PUT est envoyée à l'API pour mettre à jour les informations de l'étudiant.
  • Supprimer un Étudiant (`delete.php`): Permet de supprimer un étudiant en spécifiant son ID dans un formulaire. Une requête DELETE est envoyée à l'API pour effectuer la suppression.

Technologies Utilisées

Frontend : HTML, CSS
Backend : PHP
Communication avec l'API : cURL

Fonctionnement

  1. Lecture des Étudiants (`read.php`):
    • Récupère la liste des étudiants depuis l'API.
    • Affiche les détails des étudiants.
  2. Ajout d'un Étudiant (`create.php`):
    • Envoie une requête POST à l'API pour ajouter un nouvel étudiant.
  3. Modification d'un Étudiant (`update.php`):
    • Affiche les détails des étudiants avec des formulaires de modification.
    • Envoie une requête PUT à l'API pour mettre à jour les informations de l'étudiant.
  4. Suppression d'un Étudiant (`delete.php`):
    • Affiche un formulaire pour spécifier l'ID de l'étudiant à supprimer.
    • Envoie une requête DELETE à l'API pour supprimer l'étudiant.
