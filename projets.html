Activité 1

Formulaire d'Inscription avec Traitement PHP

I. Introduction

  • Formulaire d'inscription HTML avec des champs pour les informations personnelles.
  • Traitement côté serveur en PHP pour enregistrer les données dans une base de données.

II. Structure du Code

A. Formulaire HTML
  • Condition: Vérification si le bouton "enregistrer" a été soumis.
  • Actions:
    • Inclusion du fichier bd_connexion.php pour la connexion à la base de données.
    • Préparation d'une requête SQL d'insertion.
    • Liaison des valeurs du formulaire à la requête préparée.
    • Exécution de la requête pour enregistrer les données.
B. Traitement PHP
  • Condition: Si le bouton "enregistrer" est soumis.
  • Actions:
    • Connexion à la base de données à l'aide de la fonction connexionPDO().
    • Récupération des départements depuis la base de données pour le menu déroulant.
    • Affichage du formulaire avec des champs pour la tranche d'âge, les loisirs, et le projet de développement web.

III. Points Importants

A. Sécurité
  • Utilisation de requêtes préparées pour éviter les injections SQL.
B. Base de Données
  • La connexion à la base de données est gérée par la fonction connexionPDO() dans bd_connexion.php.
C. Affichage Conditionnel
  • Le formulaire est affiché ou le traitement est effectué en fonction de l'action de l'utilisateur.
IV. Conclusion
  • Un formulaire complet d'inscription avec une logique de traitement côté serveur pour l'enregistrement des données.

Projet PHP Orienté Objet - Gestion d'Étudiants et Promotions

Classes Principales

Classe Etudiant

Attributs privés :

  • Code : Identifiant unique de l'étudiant.
  • Nom : Nom de l'étudiant.
  • Prenom : Prénom de l'étudiant.
  • laPromo : Objet de type Promotion représentant la promotion à laquelle l'étudiant appartient.
Méthodes :
  • Constructeur : Initialise les attributs de l'étudiant.
  • addEtudiant() : Ajoute l'étudiant à la base de données.
  • ... : D'autres méthodes peuvent être ajoutées pour les fonctionnalités spécifiques.

Classe Promotion

Attributs privés :

  • Code : Identifiant unique de la promotion.
  • Nom : Nom de la promotion.
  • etudiantCollection : Tableau d'étudiants appartenant à la promotion.
Méthodes :
  • Constructeur : Initialise les attributs de la promotion.
  • getUnePromo($code) : Récupère les détails d'une promotion avec les étudiants associés.
  • ... : D'autres méthodes peuvent être ajoutées pour les fonctionnalités spécifiques.

Utilisation dans le Script PHP

Ajout d'un Étudiant

        
$promo = new Promotion('B2SIO', 'Classe de BTS SIO 2ème année');
$etudiant = new Etudiant('E7', 'Petit', 'Simon', $promo);
$etudiant->addEtudiant();

Récupération d'Étudiants et de Promotions

        
$etudiant = new Etudiant();
$etudiants = $etudiant->getEtudiants();

$promo = new Promotion();
$promoDetails = $promo->getUnePromo('B2SIO');

Remarques

  • Assurez-vous d'avoir une structure de base de données cohérente avec les requêtes SQL utilisées dans les méthodes.
  • Les classes sont conçues pour encapsuler les données et les opérations liées aux étudiants et aux promotions.
  • Les méthodes des classes peuvent être adaptées en fonction des exigences spécifiques du projet.

API REST - Gestion des Étudiants

But du Projet

Ce projet consiste en une API REST développée en PHP permettant la gestion des étudiants. Elle offre des fonctionnalités CRUD (Create, Read, Update, Delete) pour interagir avec une base de données d'étudiants.

Endpoints Disponibles

  • POST /etudiant.php : Ajouter un nouvel étudiant.
  • GET /etudiant.php : Récupérer tous les étudiants.
  • GET /etudiant.php?id=X : Récupérer un étudiant spécifique (remplacez X par l'ID de l'étudiant).
  • DELETE /etudiant.php?id=X : Supprimer un étudiant spécifique (remplacez X par l'ID de l'étudiant).
  • PUT /etudiant.php?id=X : Mettre à jour un étudiant spécifique (remplacez X par l'ID de l'étudiant).

Utilisation

Pour utiliser cette API, vous pouvez envoyer des requêtes HTTP à ses différents endpoints en utilisant des outils comme Postman, cURL, ou même depuis votre propre application.

Exemple d'utilisation avec Postman

  1. Ajouter un étudiant : POST /etudiant.php avec les données appropriées dans le corps de la requête.
  2. Récupérer tous les étudiants : GET /etudiant.php.
  3. Récupérer un étudiant spécifique : GET /etudiant.php?id=X.
  4. Supprimer un étudiant : DELETE /etudiant.php?id=X.
  5. Mettre à jour un étudiant : PUT /etudiant.php?id=X avec les nouvelles données dans le corps de la requête.

Remarques

  • Ce projet suppose l'utilisation d'une base de données configurée selon les besoins de l'API.
  • Assurez-vous d'avoir les autorisations nécessaires pour effectuer des opérations CRUD.
  • Les réponses de l'API sont au format JSON.
